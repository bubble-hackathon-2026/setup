# Bubble Design Tab — Context for Your Other Agent

Below is the plain-text context for the Design (element builder) tab. This is the most visually rich surface in the editor — the drag-and-drop canvas for composing pages.

---

## Layout anatomy (4-column shell)

Same outer chrome as Workflows (icon rail + top menu bar), but the middle layout is:

- **Icon rail (40px)** — Persistent mode switcher.
- **Left panel (~240px)** — Two stacked sections: (1) **Elements Tree** at the top with "Builder"/"Responsive" toggle buttons and a collapse icon, (2) **Element library** below with searchable lists of Visual Elements, Containers, Input Forms, and Reusable Elements. Element rows are 32px tall with a 16px icon + label.
- **Canvas (flex, light-gray bg `#F7F7F7` around a white page)** — Shows the actual page as it will render, with cobalt-70 selection outlines and red measurement tags showing pixel dimensions (e.g. "1080px", "250px") in a tiny 10px red text at the edges of hovered/selected elements.
- **Right panel (~270px)** — Properties with three tabs: **Visual**, **Interaction**, **Conditional**. Same 4px radius, 12px type system as the Workflows panel.

## Top menu bar additions (Design-specific)
- **Zoom dropdown** (`70%`, `50%`, etc.) sits just left of the search icon.
- **View mode icon** (cube/3D) next to zoom — toggles between edit and visual preview modes.
- **Issue count** ⚠ with red badge (e.g. "6") — clicking opens a panel of layout/binding problems.
- **Preview** is a split button: main action launches the preview app; caret opens options (different breakpoints, debug mode, etc.).

## The three right-panel tabs

### Visual tab (appearance + layout)
Sections, always in this order, each collapsible with a triangle caret:

1. **Content** — Element-specific. For inputs: Placeholder text, Auto-binding toggle, Initial content. For alerts: Text. For file uploaders: Placeholder, Upload button with a lightning-bolt (dynamic data) icon on the right.
2. **Size** — Width & Height each have a two-field layout: a **mode dropdown** (`Fixed`/`Fit`/`Fill`/`%`) paired with a **numeric input + unit dropdown** (`px`/`%`). For Fit mode, Min/Max height reveal.
3. **Layout** — **Position** uses a 3×3 grid picker (9 dots) for alignment, with two dropdowns (`Top`/`Left`/etc.) beside it. **Order** has 4 icon buttons: Make first (⏮), Previous (↑), Next (↓), Make last (⏭). **Spacing** has Padding and Margin, each with paired inputs (vertical, horizontal) with small direction icons, plus an expand-to-4-sides icon on the right.
4. **Configure** — Element-specific binary toggles (e.g. "Position the alert at the top").
5. **Style** — A single row showing the current style name (e.g. "Standard") with an `Overridden` caption badge (10px/16px 600 weight, livid-70 `#415162`) when the style has been modified, plus pencil (edit) and chain-link (unlink/reset) icons on the right.
6. **Text** — Font dropdown (`App Font (Inter)`), weight dropdown (`400`), size dropdown (`16px`), then 3 alignment icons (left/center/right), 3 style icons (B/I/U), and a `⋯` overflow that opens a popover titled "Text Properties" with Word/Line/Letter spacing dropdowns. Color row uses a **named swatch chip** (e.g. "Text" with a 14×14 1px-radius black square) + lightning-bolt dynamic icon.
7. **Placeholder** — Same swatch+dynamic pattern, typically "Gray 40".
8. **Appearance** — Opacity (draggable input with `⋮⋮` grip on the left, `%` suffix), Radius (small corner icon prefix, expand-to-4-corners icon on the right).
9. **Border** — Collapsed by default; `+` / `−` affordance in the section header. When expanded: style dropdown (`Solid`/`Dashed`/etc.), width input with px, expand-to-4-sides icon, color swatch.
10. **Background** — Swatch with named color (e.g. "Primary 20" or hex "#FFFFFF") with opacity %, lightning-bolt for dynamic.
11. **Shadow** — Collapsed by default with `+`.

### Interaction tab
1. **Workflows** — Shows "No workflows added yet." empty state with an "Add workflow (⌘+K)" secondary button (bordered, full-width). Once added, lists workflows linked to this element.
2. **Visibility** — "Visible on page load" toggle (cobalt-70 when on), "Collapse when hidden" toggle, with info icons (ⓘ) on labels.
3. **Options** — Element-specific toggles: "Submit on 'Enter'", "Make required", "Make disabled".
4. **Transitions** — List with `+` to add. Each row shows property name (e.g. "Border color") and the duration/easing (e.g. "200, ease") right-aligned in gray-60.
5. **Advanced** — "ID Attribute" textbox for giving the element a CSS id.

### Conditional tab
- Header "All conditionals" with expand icon + `+` icon.
- "N inherited from Style" collapsible row at top (shows how many conditions come from the shared style).
- Empty state: centered illustration (a gray-20 rounded-square with `</>` icon), title "No conditionals yet", 2-line helper text, ghost button `+ Add a conditional`.

## Page-level properties
When nothing is selected, the right panel shows the page itself. Layout section reveals the signature Bubble flex controls:
- **Container layout** — 4-option segmented control with icons: `↓ Column` | `→ Row` | `⊚ Align` | `⊞ Fixed`. Active option has white background on a gray-10 track.
- **Container alignment** — 5 icons showing Top/Center/Bottom × Left/Center (for Column mode; icons change based on container type).
- **Gap** — single numeric input with a small gap-icon prefix.
- **Configure** — Page folder dropdown, "Make this page Native Mobile" toggle, Mobile version selector.

## What users can do in the Design tab

**Add elements**
- Drag from the left library onto the canvas — snap guides appear as blue dashed lines.
- Search the library via the top search field ("Search elements").
- Use "Install More" at the bottom of each library section to browse the plugin marketplace.
- Create **Reusable elements** (custom components) via "+ New Reusable" at the very bottom.

**Select & arrange**
- Click any element on canvas to select it; selected state shows a cobalt-70 outline + a tiny blue "Element Name" tag floating above its top-left.
- Red measurement labels appear showing pixel dimensions while selected.
- Click empty space (or the page) to select the page root.
- Click + Shift to add to selection (inferred standard pattern).
- Drag elements directly on canvas to reposition (when not in a fixed container).

**Right-click context menu** (rich, 18+ options in observed state)
- **Delete** (with `delete` keyboard shortcut shown right-aligned)
- **Replace the element type** — swap one element for another without rebuilding bindings
- **Convert to a reusable element** — turn a selection into a component
- **Start/Edit workflow** (⌘+K) — jumps directly to Workflows tab for this element
- **Group elements in** — submenu with container types (Group, RepeatingGroup, etc.)
- **Cut / Copy / Copy to another app / Paste** (all with ⌘+X/C/V shortcuts)
- **Copy special / Paste special** — submenus for partial copies (styles only, etc.)
- **Select all** (⌘+A)
- **Select first parent** — navigates up the tree
- **Bring to front / Send to back**
- Additional "See reference" button appears as a dark floating chip next to the menu — opens docs for the element.

**Style elements**
- Every color input accepts: hex code, named swatch from the style system, or a dynamic expression (via the lightning-bolt `⚡` icon).
- Every numeric input accepts direct typing, drag-to-scrub (inferred), or dropdown presets.
- Typography is unified: Font + Weight + Size + Align + B/I/U + overflow for advanced text properties — identical across all text-bearing elements.
- Expand-to-4-sides icon (`⧉` style icon) next to padding/margin/radius/border opens an expanded view where each side can be set independently.
- The chain-link icon next to Style name unlinks the element from its shared style, converting styles to inline.

**Responsive design**
- Top-of-sidebar "Builder | Responsive" toggle switches mode.
- Responsive mode adds a top strip of breakpoint tabs: `1200 | ★ Default | 992 | 768 | 320 | +`, where ★ denotes the base breakpoint and `+` adds custom ones.
- Bottom-left of the sidebar gains a live debug inspector showing: `Page 1080px`, `Parent container 1080px`, `Element 250px` with cobalt-70 numbers, plus a **Page states** section that lets you toggle simulated states like "User logged out".

**Inspect & navigate**
- The top menu's `Find element...` search field (magnifier icon) jumps to any element by name.
- Elements Tree in the left panel mirrors the canvas as a collapsible tree — click to select, drag to reparent.
- The breadcrumb in the top bar updates to show the selected element name (e.g. "Input A" with an X to close selection).
- Tooltips appear on hover for all icon-only controls (Design, Workflow, Builder/Responsive, etc.).

## Signature visual patterns (Design-tab-specific)

**Color picker popover**
When you click any color swatch:
- Title bar with the section name ("Background", "Border", "Text") + close X.
- 4-tab icon row: solid fill, gradient (stripes icon), image (picture icon), video (play icon).
- Classic HSL saturation/lightness square (~240×160), hue slider below, opacity slider below that.
- Hex input + opacity % field at the bottom.
- **"Variables" section** with a list-view toggle and `+` to add — displays a **5-column × 7-row grid of ~24×24 swatches** in cobalt / white / black / red / green / yellow rows, each row going light-to-dark. Variables have a search field above the grid.
- Popover itself: white bg, 4px radius, `--b-popover-shadow`, ~250px wide.

**Named color chip pattern** (the core Bubble styling convention)
- 14×14 swatch with 1px radius (almost square!)
- Paired with a color variable name to its right ("Text", "Primary 20", "Gray 40", "Success 20")
- Followed on the right by a purple lightning-bolt icon (⚡) that triggers the dynamic data picker
- This is THE way colors are chosen in Bubble — direct hex is an escape hatch, named variables are the norm

**Segmented controls**
- Inactive segment: gray-20 (`#F2F2F2`) track, gray-80 text
- Active segment: white pill inside track (subtle shadow), gray-80 text, slightly bolder
- Icon-first with 12px label below OR icon-only (for alignment 5-icon rows)
- 4px outer radius, 2–3px inner pill radius

**Selection overlay on canvas**
- 1px cobalt-70 solid outline around selected element
- 8×8 cobalt-70 drag handles at corners and midpoints (inferred standard)
- Small cobalt-70 label tab above top-left: dark blue fill, white text, 10px/16px 400 weight, 4px horizontal padding, 2px radius — reads the element type/name (e.g. "⇶ Input A")
- Red pixel-measurement captions at element edges in tiny 10px red-70 text (non-selectable UI chrome)

**Empty states (consistent template)**
- Centered illustration: a rounded-square panel in gray-20 with a simple monochrome icon (e.g. `</>`), ~80–100px
- 12px/16px 700 gray-80 headline
- 12px/16px 400 gray-70 2-line helper text, center-aligned, max ~260px wide
- Ghost button at bottom: `+` icon + label, cobalt-70 text on transparent bg, hovers to gray-10

**"Make dynamic" lightning bolt**
- 16×16 purple (`#9606BC`) thunder icon, clickable, sits flush right of any text/number/color input that accepts dynamic expressions
- When clicked, converts the input into an expression builder (same dynamic data picker as Workflows)
- Once a dynamic expression is set, the input shows a purple-20 bg with purple-70 text in token-chip form

**Style system linkage**
- Any element linked to a shared style shows a "Style" section with the style name + pencil + chain icons.
- Pencil = edit the shared style (jumps to Styles tab).
- Chain (two link icons, chain-broken visual) = unlink this element's styles from the shared style.
- When any property has been changed locally, a small "Overridden" caption appears in livid-70 (`#415162`) 10px/16px 600 weight next to the style name.

## UX principles (Design tab additions to earlier list)

Beyond the workflow-tab principles:

1. **Every numeric has a unit.** Inputs show `px`/`%`/`em` as a dropdown suffix, never inferred — this makes the editor feel precise and mirrors CSS mental model.
2. **Named variables over raw values.** Color system, font system, spacing system all reward naming. Raw hex/px is available but visually de-emphasized.
3. **Dynamic data is purple.** Across the entire editor: any value that can come from runtime data uses purple (⚡ icon, chip background, token styling). Static values stay in gray/cobalt.
4. **Expand-where-you-need.** Paired inputs (margin, padding, radius, border) default to a simple 2-input view but each has an "expand to 4 sides" icon for fine-grained control.
5. **Selection scales to context.** Selecting an element, a group, or the page itself all use the same right-panel shell — no mode switches, just different available sections.
6. **Preview always one click away.** The Deploy button is always visible, but so is Preview (with its own dropdown for breakpoint/debug preview modes). Editing and testing stay in constant contact.
7. **Red for geometry, cobalt for selection.** Red pixel labels communicate "this is a measurement, not content"; cobalt outlines communicate "this is selected." Never swapped.

## Canvas design details

- Canvas background: soft `#F7F7F7` gutter around a pure white `#FFFFFF` page.
- Page has a 1px livid-20 (`#E4EAF1`) outline to show page bounds.
- Table/grid elements render with 1px gray-30 cell borders.
- Snap guide lines (when dragging) appear as 1px cobalt-70 dashed lines.
- Text elements show a subtle "edit me..." placeholder in italic gray-70 when empty.
- Links show as cobalt-70 text with underline by default.
- Alert elements have rounded 4px bg tint matching their semantic type (green bg for success alert, etc.) with darker text in same hue.
- Slider has cobalt-70 filled portion, cobalt-20 track, cobalt-70 square handle.
- Radio buttons are cobalt-70 filled circle when selected, gray-40 border when empty.
- Checkbox has gray-40 1px border, cobalt-70 fill with white checkmark when checked.
- File uploader is a button-style pill: white bg, gray-30 border, "Click to upload a file" label.
- Picture uploader is a bordered square placeholder with "Click to upload an image" centered text.

## Keyboard shortcuts observed in Design tab
- `delete` — delete selected element
- `⌘+K` — start/edit workflow for selected element
- `⌘+X` / `⌘+C` / `⌘+V` — cut/copy/paste
- `⌘+A` — select all
- `⌘+F` (via search icon) — find element
- `⌘+Z` / `⇧⌘+Z` — undo/redo (top bar)
- `Escape` — close popovers, deselect
