# Bubble Design Context

This folder contains reference materials for building prototypes that look and feel like Bubble's product.

## What Goes Here

- **`bubble-overview.md`** — Product overview, design tokens (colors, fonts, spacing), terminology
- **`screenshots/`** — Screenshots of the Bubble editor and key UI patterns

## For Contributors (LD, Mirea, Design Team)

Add materials here that would help someone (or an AI) recreate Bubble's UI:

1. **Screenshots**: PNG/JPG files in `screenshots/`. Name them descriptively:
   - `editor-canvas.png` — The main editor canvas view
   - `expression-editor.png` — The expression/formula editor
   - `property-panel.png` — The property panel / inspector
   - `workflow-tab.png` — The workflow editor
   - etc.

2. **Design tokens**: Update `bubble-overview.md` with:
   - Primary colors and their hex values
   - Font families and sizes
   - Spacing scale
   - Border radius conventions
   - Icon style (Phosphor icons, weight, sizing)

3. **Component patterns**: Describe common UI patterns:
   - How dropdowns look and behave
   - Panel/sidebar layout conventions
   - Button styles (primary, secondary, ghost)
   - Form input styling

The more specific and visual the context, the better the prototypes will look.
